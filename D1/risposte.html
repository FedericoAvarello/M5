1 Cos’è ReactJS? ReactJS è una libreria JavaScript per la creazione di interfacce utente, sviluppata da Facebook. È utilizzata per la costruzione di componenti riutilizzabili e dinamici, che consentono di creare interfacce utente interattive e reattive. 2 ReactJS è un framework, non una libreria. (v/f) Falso. ReactJS è una libreria JavaScript, non un framework completo. È focalizzato sulla creazione di componenti riutilizzabili per la gestione dell'interfaccia utente. 3 Il file package.json contiene… Molte informazioni utili, come ad esempio l’elenco di tutte le dipendenze richieste dall’applicazione. 4 create-react-app è l’unico modo possibile per creare un’applicazione React (v/f) Falso. Create React App è uno strumento popolare per creare rapidamente un'applicazione React con una configurazione predefinita. Tuttavia, esistono anche altri modi per configurare un'applicazione React, come l'utilizzo di strumenti come Webpack o Parcel da zero. 5 Qual è il comando da lanciare nel terminale per creare una nuova create-react-app con nome "test"? Il comando corretto è: npx create-react-app test 6 Cos’è un componente React? Un componente React è un blocco di logica/contenuto riutilizzabile all’interno dell’applicazione. È una parte indipendente e isolata dell'interfaccia utente che può essere riutilizzata in diverse parti dell'applicazione. 7 Un componente React può venire creato in tre modi: come funzione, classe o interfaccia. (v/f) Falso. Un componente React può essere creato come funzione o come classe, non come interfaccia. 8 Qual è la differenza tra componenti creati come funzione e componenti creati come classe? I componenti creati come funzione sono più semplici da scrivere e hanno meno funzionalità rispetto ai componenti creati come classe. I componenti creati come classe richiedono generalmente più linee di codice, ma offrono funzionalità aggiuntive come lo stato interno e i metodi di ciclo di vita. 9 Le props sono frammenti di informazione assegnati all’invocazione di un componente React, utili al fine di rendere il componente dinamico e più riutilizzabile. (v/f) Vero. Le props sono i meccanismi principali per passare dati da un componente genitore a un componente figlio. Consentono di rendere un componente React più dinamico e riutilizzabile. 10 Le props possono essere passate solamente da un componente genitore a un componente figlio, non è possibile fare il contrario (v/f) Vero. Le props vengono passate da un componente genitore a un componente figlio e non possono essere trasmesse in modo diretto in direzione inversa. 11 Da dove possono venire recuperate le props all’interno di un componente React creato come classe? Le props possono essere recuperate all'interno del componente React creato come classe attraverso l'oggetto 'this', dentro un sotto-oggetto chiamato 'props'.
